# STM32G431 高性能硬核穿越机飞控 (Flight Controller)

<div align="center">
  <p>🚀 基于 STM32G431CB 打造的硬核多旋翼控制算法系统（纯 C 语言 / 裸机硬实时架构）</p>

## 📖 项目简介
本项目是一个基于 STM32G431 平台、从零手写的**硬核多旋翼飞行控制器固件**。

它没有使用任何繁重的 RTOS（实时操作系统），而是汲取了现代 FPV 极限竞速飞控（如 Betaflight）的底层精髓。通过**纯硬件中断（DRDY）**、**DMA 数据搬运**以及**极简的时间片微调度器**，构建了一个高达 **2000Hz (0.5ms)** 刷新率的硬实时串级 PID 控制闭环大动脉。

这里面抛弃了为了兼容各种机架而写的数百万行臃肿代码，只提纯了能让四轴飞行器违背引力、死死钉在空中的最高阶算法：双二阶动态陷波滤波（FFT Dynamic Notch）、Anti-Windup 抗积分饱和、D-Term LPF 滤波、I-Term Relax 放松机制，以及卡尔曼物理定高预测。

## 🛠️ 硬件架构选型

本项目精简了繁杂的外围设备，专注于核心飞行性能的极致压榨：

*   **MCU 主控芯片:** 意法半导体 **STM32G431CB** (Cortex-M4, 170MHz 主频, 内置 FPU 浮点运算单元与硬件数学加速器 CORDIC)。
*   **核心 IMU 惯导单元:** 
    *   TDK **ICM42688-P** (世界上最顶级的极低底噪 6 轴陀螺仪/加速度计之一)。
    *   **通讯:** 使用极速 **SPI 接口**连接。
    *   **时钟源:** 动用引脚的 **EXTI 外部中断 (DRDY Data Ready)**，让传感器亲自敲打单片机的门，作为整个飞控系统跳动的总时钟源。
*   **气压计 (Barometer):** TE Connectivity **MS5611**，通过 **I2C 接口**连接，提供高精度的相对海拔气压数据，分辨率可达 10 厘米级别。
*   **电调/电机输出:** 
    *   支持传统模拟 **PWM** (400Hz 刷新率, 定时器 TIM 挂载)。
    *   深度支持了目前最前沿的 **DShot600 数控协议** (600kbit/s 纯数字协议无延迟通讯，利用 DMA+TIM 混合调制硬编码)。
*   **通信射频 (Receiver):** 支持串口接收机协议 (SBUS/CRSF 等) 以及原生的定时器输入捕获 PPM 解码。

## ⚙️ 核心控制算法与黑科技特性

由于这是一份极具技术含量的硬核固件，其主要实现了以下惊艳控制层的技术降维打击：

### 1. ⚡ 纯硬实时驱动架构 (Hard Real-Time Loop)
*   **抛弃阻塞轮询：** 终结了业余级代码里泛滥的阻塞 `HAL_Delay()` 控制。
*   **中断为主，调度为辅：** 传感器 ICM42688 内部高精度晶振每 500 微秒发出一声嘀嗒（引脚跳变），触发 STM32 的极高优先级外部中断。在中断服务区 (ISR) 配合 DMA 瞬间完成 SPI 数传，并原地执行 `FlightControl()`。确立飞控“大脑心跳 (`dt`)” 在 0.005 秒上误差卡死在微秒级，这是算出绝对精确姿态数学积分的物理基础。

### 2. 🧮 现代飞控姿态解算模块
*   深度植入了 **Madgwick / Mahony 四元数更新算法**。
*   拒绝了传统的欧拉角计算，防止了在仰角 90 度时的死锁现象 (Gimbal Lock)；巧妙利用重力加速度作为绝对参考系向量，使用缓慢补偿收敛技术（PI 项），持续纠正陀螺仪温度漂移产生的误差。

### 3. 🎯 高阶非线性“串级 PID”防御控制
并未采用粗暴单薄的“单环 PID”，而是标准且复杂的外环（角度寻地）+ 内环（角速度抗矩）串级控制，且融入了四大高级功能：
*   **Anti-Windup (抗积分饱和机制):** 当机架卡树或是全盘满负荷时，算法自动探测（Pre-Output）边界，强力切断并冻结积分项 (I-Term) 瞎累加，防止脱困瞬间引发的疯狂“死亡翻滚”。
*   **TPA (油门 PID 衰减机制):** 随着用户暴力推油（大于 50% 阈值即进入高风阻狂飙期），自适应按比例削弱 P 项和 D 项力量，防止高速抽搐。
*   **Anti-Gravity (反重力模式保护):** 监测用户迅速推拉油门杆的瞬间动量变化，以数学倍率临时翻倍强化 I 项（积分锁死强度），让机鼻在垂直猛抛机动中宛如死死锁在空气中不低头点车。
*   **I-Term Relax (积分休眠):** 用户猛打摇杆翻滚时，主动麻痹积分，解决翻滚归中结束瞬间那令人恶心的“最后弹簧回弹”（Bounce Back）效应。

### 4. 🔪 工业级数字信号处理器 (DSP FFT Filters)
当四列狂躁无刷电机加上廉价塑料螺旋桨产生巨大破空共振（甚至高达 100~300Hz）时，剧烈的机械共振对 PID D项是毁灭性的打击。本项目加入了完整严苛的多级信号滤波战壕：
*   **一阶低通滤波防线 (PT1 / PT2 LPF):** 互补指数低通滤波器，强制没收 90Hz 以上的随机高频锯齿噪音信号，剥离出相对平滑顺畅的数据基线。
*   **最恐怖的神技：基于快速傅里叶变换的动态陷波滤波器 (Dynamic FFT Notch Filter)**。
    STM32 芯片在 2kHz 的超速主闭环之外的微小缝隙里，由后台 50Hz 的任务片唤醒底层 ARM-DSP 加速库。将 256 位时域生肉数据强行转化成**频域频谱图 (FFT)**，**并找出当前此刻能量最高的那根“毒刺”频率，随后计算出一组精确度极高的双二阶（Biquad IIR）数字阻滞缺口，像手术刀一般彻底抹除该特定频率！而且它会随着用户的转速加速而动态跟踪**。这项技术的引入，终结了电机产生致命共振烧毁的疑难杂症。

### 5. 🚁 异构传感数据融和（定高算法 Alt-Hold）
为了对抗气压计先天的“慢”（50Hz获取，且读数随风起伏波动），项目利用了**卡尔曼滤波 (Kalman Filter)** 的预测-观测思想雏形：
*   **预测步：** 以 2000Hz 超高速频率运行的 Z轴 加速度计计算重力分量积分，作为运动学的高维推演流。
*   **修正步：** 将缓慢且绝对位置准确的气压计数据混入其中，以消除纯加速度积分那无法避免的长时漂移问题。最终产出稳定的 3D 定位坐标供位置 PID 使用。

### 6. 🛡️ Failsafe 物理断头台与看门狗
*   **强硬级失控保护:** 接收机端如发生信号失锁或数据包校验失败，计时器累计 0.5 秒直接封死脉冲输出并关闭动力。
*   **底层独立看门狗芯片 (IWDG):** 系统复位时点燃 100 毫秒炸药包。只要后台极不重要的一项任务中途出现死循环（例如恶劣 I2C 总线假死抱死），导致超 100 毫秒无法前往喂狗（Refresh），单片机会无情地产生极高引脚硬件层 Reset，让僵死系统立刻起死回生挽救飞行器。
